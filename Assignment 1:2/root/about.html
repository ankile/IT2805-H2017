Questions

Svar på spørsmål:

  1. In the client-server architecture, the server, as the name might imply, serves clients, or in the case of the web, people who want to access the information on a web page, with said information, after a client has requested it. A server will often serve many clients at a time.
  2. The client sends a request to the server, where it asks for the file or piece of information it wants, and waits for a response from the server. An example of a client might be my phone when I go to youkok.com, it sends a request to the server, and recieves løsningsforslag.
  3. An IP adress, or an Internet Protocol Adress, is a unique number assigned to every device on the network, such as PCs, phones or printers. The adress is used to identify different devices on the network
  4. DNS, or Domain Name System, translates the easy-to-remember URLs we use every day, into the long, numerical IP adresses, which are are the actual adress of a website.
  5. The internet is the enormous web of connected devices that lets any device speak to any other as long as they're both connected to the internet. The World Wide Web, on the other hand, is a way of sharing information over the web, built on top of the internet. The WWW uses the HTTP protocol for information transfer, which is just one of the languages that could be used for such things.
  6. HTML, or Hyper Text Markup Language, is the basic building block of any webpage. It structures the content using different elements, like the one I'm using here to create the numbered list.
